# 次回セッション 改善メモ
> 作成: 2026-03-30

---

## 1. FLUTEとTURINGのパラメーター効きが良くない

**現状の問題:**
- ドローン（持続音）は変化が出にくい音の性格上、パラメーター変化が聞き取りにくい
- CHAOSがfeedback_amtとshimmerにマップされているが、変化が地味
- TURINGの音程変化は8小節に1回でゆっくりすぎる

**改善アイデア:**
- FLUTEのCHAOSをもっと派手なパラメーター（amp変調・ピッチ揺れ）にもマップ
- TURINGのmutation_barsをGUIから調整できるようにする（速さを変えられる）
- パラメーター変化に「ビジュアルフィードバック」を追加してどこが動いているか見えるように

## 2. ドローンが暗い

**現状の問題:**
- LocalIn/LocalOut削除後、高調波が減って全体的に暗い
- BRIGHTNESSスライダーは追加済みだが、デフォルト値が低め

**改善アイデア:**
- BRIGHTNESSのデフォルトを0.5 → 0.65に上げる
- Klankの倍音構成を見直す（より高い倍音を強調）
- 明るい/暗いを切り替えるプリセット（Tone: Bright / Dark / Warm）を追加
- GrayNoise励振への切り替えオプション（より金属的・明るい質感）

## 3. リズム音がチープ・もっとAutechre/池田良二/Alva Noto的な電子音にしたい

**目標サウンド:**
- シンセとドラムの中間
- パルス波のいろんなパターン
- Autechre: 変形したリズム・非整数倍音のパーカッション
- 池田良二: サイン波・クリック・純粋な電子音
- Alva Noto: グリッチ・デジタルノイズ・建築的反復

**改善アイデア:**
- KlankやRingzベースの「共鳴パーカッション」（金属板を叩くような音）
- DynKlank（倍音が動的に変化）をリズム音源に使う
- サイン波 + 量子化ノイズ（Alva Noto的グリッチ）
- WhiteNoise + 超短いBPF（池田良二的クリック）
- 音長・ピッチをリズム毎にランダム変化させる（単調にならない）
- 参照ファイル: `knowledge/rag/sources/artist_autechre_production.md`
           `knowledge/rag/sources/artist_alvanoto_aesthetics.md`
           `knowledge/rag/sources/sc_exciter_types.md`

## 4. ステップシーケンサーがシンプルすぎる

**現状の問題:**
- 16ステップの固定グリッドは機械的・規則的になりすぎる

**改善アイデア:**
- **確率モード**: 各ステップに発火確率（0〜100%）を設定できる（すでにSCには実装済み・UIのみ）
- **ランダマイズ**: ボタン1つでパターンをランダム生成（密度を指定）
- **スライス**: 既存パターンをランダムに入れ替える（シャッフル）
- **ユークリッドリズム**: EUCLボタンは実装済み・もっと前面に出す
- **ポリリズム**: トラックごとに異なるステップ数（例: 12/16/13ステップ）
- **グリッドレス**: BPMに依存しない時間ベースのトリガー（ポアソン過程）
  → CHAOS ENGINE（すでにSCに実装済み）をUIから制御可能にする
- 参照ファイル: `knowledge/rag/sources/design_probability_control.md`
           `knowledge/rag/sources/sc_livecoding_techniques.md`

## 5. 音のバリエーションを増やしたい（触覚の多様性）

**目標:** ドローンに加えて、短い・動く・メロディっぽい音を追加

**改善アイデア:**

### アルペジエーター
- TURINGの音程シーケンスを使ったアルペジエーター
- `/matoma/arp/` OSCで制御
- rate / octave / pattern (up/down/random)

### メロディシンセ
- FMシンセ（matoma_basicを流用）をシーケンスで鳴らす
- 短いノート（スタッカート）の音列を生成

### グラニュラーをメロディっぽく
- 録音バッファのポジション（pos）をシーケンスで動かす
- 特定音程の部分だけ再生する「ピッチトラッキンググラニュラー」

### テクスチャレイヤー
- GrainBufを使った「動く砂粒のような」テクスチャ
- Autechreの「分解されたビート」感

### 参照ファイル

- `knowledge/rag/sources/artist_autechre_production.md`

## 6. グリッチ

**目標:** デジタル的な壊れ・欠陥音を音楽的素材として使う

**改善アイデア:**
- **ビットクラッシュ**: `Latch` + 低サンプルレートで量子化ノイズ
- **バッファスクラブ**: グラニュラルのposを突然ジャンプさせる
- **スペクトルグリッチ**: FFT → `PV_BrickWall` / `PV_RandComb` で周波数を破壊
- **リングモジュレーション**: 音同士をかけ合わせて金属的な歪み
- **グリッチシーケンス**: ランダムなタイミングで一瞬音が「壊れる」イベント
- GUIに「GLITCH」ボタンを追加 → 確率・強度を制御

- 参照ファイル: `knowledge/rag/sources/artist_alvanoto_aesthetics.md`（失敗の美学）
           `knowledge/rag/sources/synthesis_spectral.md`

## 7. 音楽的調和の基盤（最重要・次回最優先）

**現状の問題:**
現在はドローン・グリッチ・リズムがそれぞれ独立してパラメーターを動かしており、
「ピュアなノイズの集合体」になっている。音楽的な調和・統合性がない。

**SPEC.mdとの対応（階層的時間制御）:**
```
Song Form（数分）  ← Tidal が担う → 未実装
Section（8〜32小節）← シーン切り替え → 実装済み（弱い）
Phrase（1〜4小節） ← 半自動エンベロープ → 未実装
Parameter（ms〜秒）← ChaosEngine → 実装済み
```

**必要な実装（優先順）:**

### 7-1. 共通スケール制約（全音源が同じ音階を共有）
- TURINGが使っているスケール（warm/void/vast/lost のDNA）をドローン・グリッチにも適用
- グリッチ(BURST)のRingz周波数をスケール音程に制約
- 実装: `~currentScale` グローバル変数でスケール共有

### 7-2. コードプログレッション（Phraseレベルの調和）
- 4〜8小節周期で和音の中心が移動する
- TuringGeneを拡張して「コード進行シーケンス」を持たせる
- ドローンのfreqがこの進行に追従する
- 参照: knowledge/rag/sources/generative_music_theory.md

### 7-3. 相対的カオス（大きいカオス）の実装
- 「予定された調和」があって初めて「崩し」が意味を持つ
- FLUTEのTENSION → 調和の崩れ度合い（スケール外に出る確率）
- CHAOSが高いほど和音の境界が曖昧になる
- これがライブ全体の「緊張と解放」を作る

### 7-4. TidalCycles統合（Song Formレベル）
- コード進行・リズムパターンのOSC制御
- bridge.pyのtidal関連コードが未完成 → 完成させる
- 参照: knowledge/rag/sources/tidal_pattern_functions.md

**設計の核心（SPEC.mdより）:**
> あなたは嵐の進路を決める。嵐の形は自然が決める。
> ジャズのコード進行と同じ原理 — コードは固定、即興は自由、調性は守られる。

**優先順位:**
| 優先 | タスク | 効果 |
|------|--------|------|
| 🔴 最高 | 共通スケール制約の実装 | 全音源が調和する |
| 🔴 最高 | コードプログレッション | 音楽的な動きが生まれる |
| 🟡 高 | 相対的カオスパラメーター | ライブの展開を作れる |
| 🟡 高 | TidalCycles統合完成 | Song Formレベルの制御 |
