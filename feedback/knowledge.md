- `GrainSin`（バッファ不要）は `GrainBuf` より安定して起動できる

- `Env.adsr(2.0, ...)` は通常使用では自然だが、テスト時は `attack: 0.3` に短縮すること

| 状況 | 原因 | 対策 |
|------|------|------|

| ChaosEngine がスライダー操作を上書き | 10回/秒でパラメーターを送り続ける | ChaosEngine の `_DEFAULT_CHAOS_PARAMS` に追加しない |

## 設計判断ログ

> 「良くなった」確認後にClaudeが記録する。フォーマット: 判断・なぜ・使う状況・使ってはいけない状況。

### MaToMaの設計哲学：羊飼いモデル（2026-03-30）

**判断**: MaToMaのプレーヤーは「演奏家」でも「庭師」でもなく「羊飼い」として設計する。

**なぜ**: 音が自律的に動く意志を持っている前提から始まる。プレーヤーは群れの方向を少し修正するだけで、一匹一匹の動きは制御しない。庭師は植えた植物に責任を持つが、羊飼いは羊が羊として動く自由を信頼する。

**羊飼いの道具（許容される操作）**:
- 犬 = シーン切り替え（群れの方向を変える）
- 牧草地の境界 = Limiter・Clip（安全な範囲）
- 笛 = 少数のスライダー（全体への合図）

**使う状況**: 新機能を追加するかどうか判断するとき。「これは羊飼いの道具か、羊を直接操る鎖か？」で判断する。

**使ってはいけない状況**: 鎖（直接制御）が増えるほどシステムは元のシンセサイザーに戻る。パラメーターを細かく操作できる機能の追加は羊飼いモデルに反する。

---

### 物理モデリング方向への検討（2026-03-30）

**判断**: 次の合成拡張として「物理モデリング共鳴体」（Klank/DynKlank/Ringz）を検討中。Layer 0を「パラメーターを受け取る機械」から「非線形で素材感のある物体」に変える方向。

**なぜ**: 現行Layer 0〜3はすべてパラメーター変調の範疇。アーティスト参照（Autechre・Tim Hecker）が行う「合成構造そのものの変化」を実現するには合成エンジンの再設計が必要と判断。

**RAGギャップ（情報収集待ち）**:
- DynKlankの実装パターン（リアルタイム周波数更新の方法）
- 素材別周波数比テーブル（ガラス・木材・真鍮）

**使う状況**: Phase 1〜2（安全制御・Layer A）完了後、Layer 0の再設計フェーズで参照する。

**注意**: Tim Hecker的な音はKlank/Ringzより `DFM1`自己発振 + フィードバックドローンの方が近い。物理モデリングはどちらかというとAutechre的金属感になりやすい。

## 設計哲学：2種類のカオス

### カオスの2層構造

| カオスの種類 | 時間スケール | 役割 | 現在の実装 |
|------------|-----------|------|----------|
| **大きいカオス（相対的カオス）** | 数分・数十小節 | ライブ全体の展開。「予定された調和」からどれだけ崩れるかを制御 | **未実装** |
| **小さいカオス（パラメーターランダマイズ）** | ミリ秒〜秒 | 瞬間的なパラメーター変動。ChaosEngineが担当 | 実装済み |

### 実装すべき「予定された調和」の基盤

```
調和の基盤（大きいカオスが崩す対象）
├── コードプログレッション（4〜8小節で和音が移動）
├── スケール制約（音程はスケール外に出ない）
│   → ドローン・グリッチ・リズムが同じスケールを共有
├── 共通リズムパターン（グリッド・ユークリッド等）
└── シーン引力点（シーン切り替えで調和の中心が移動）
```

### 大きいカオスの設計方針

- ジャズのコード進行と同じ原理: **コードは固定、即興は自由、調性は守られる**
- TidalCyclesのパターン機能がこれを担う
- FLUTEのTENSIONを「調和の崩れ度合い」として使う設計が適切
- グリッチシンセの周波数域もスケールに制約すると音楽的統合性が高まる

| ユーザーの言葉 | SPEC.mdの対応概念 | 実装アイデア |

| 「予定された調和」 | Song Form / Section | TidalCycles コード進行 |
| 「相対的カオス（大）」 | 階層的時間制御・上位レイヤー | FLUTE TENSION / シーン崩壊パラメーター |

## 設計哲学：羊飼いメタファー（モジュール間相互作用）

> 2026-03-30 ユーザーから提示された重要な設計思想。

### 2種類の自由と制約

```
羊飼い（大きな統合）: Tidal / シーン / FLUTE マクロ
    → 1つの方向性を厳守。境界（ceiling/floor）を定義する。

羊（小さなカオス）: モジュール間の相互作用
    → ドローンとグリッチが影響し合う、リズムがグラニュラーを揺さぶる
    → 喧嘩・仲良し・距離の自由な関係
    → ただし境界の外には出られない
```

### Organic Coupling の拡張方向

現在: ドローンの明るさ（spectralCentroid）→ グラニュラーのグレイン密度
将来:
- リズムのビート → グラニュラーのpos同期
- グリッチのBURST発火 → ドローンのshimmer一時上昇
- ドローンのfeedback_amt高 → グリッチchaos連動

### BPMという概念はMaToMaでは使わない

> 2026-03-31 ユーザーの設計方針。

**BPMは「固定テンポのグリッド」を前提にした概念。MaToMaが参照するアーティスト群はそこから逃げている。**

```
Alva Noto: 1/192拍子の数学的細分割・フィボナッチ列・位相差
           → 「グリッドを使うが、BPMに縛られない」

OPN:       LFO周期（0.05〜0.1Hz = 10〜20秒）による呼吸
           → 「ビートという概念ごと消す」
```

MaToMaでリズムを扱うときの適切な単位：
- **時間（秒・Hz）**: LFO、密度変動、呼吸周期
- **比率（分数・フィボナッチ）**: 粒の出現タイミングの数学的配置
- **確率（density・Dust）**: 偶発的打突の頻度

「この音は何BPMか？」という問いを立てない。
「この音の呼吸周期は何秒か？」「どの数学的比率で配置されているか？」が正しい問い。

## パン技法（Alva Noto / Ryoji Ikeda スタイル）

> パン＝音の第3パラメータ（周波数・音量と同等）。各粒に独立した空間運動を与える。

### 固定パン（Alva Noto基本）

- Lチャンネル：−0.3 / Rチャンネル：+0.3
- 同一粒が2つの独立したリズムイベントとして機能する

### LFOパン 周波数帯域と効果

| 速度 | 周波数帯 | 効果 |
|------|---------|------|
| 低速 | 0.5〜2Hz | 呼吸感・空間拡張 |
| 中速 | 3〜5Hz | 揺らぎ・有機性 |
| 高速 | 6〜10Hz | Ikeda的空間跳躍 |

### 周波数帯域別パン原則

| 帯域 | 範囲 | パン幅 |
|------|------|-------|
| 低域 | 〜200Hz | センター固定（動かさない） |
| 中域 | 200〜2kHz | ±0.2〜0.4 |
| 高域 | 2kHz〜 | ±0.6〜1.0（極端） |

### 複数粒の空間分離（混濁回避・3Dリズム）

複数の粒を同時に配置するとき、各粒のLFO周波数をずらす。例：
- 粒1: 6.28Hz
- 粒2: 7.83Hz
- 粒3: 4.9Hz
→ 互いに干渉せず3D空間に広がる

### 位相オフセット技法

同じLFO周波数でも位相差をつけると複雑な空間軌道が生まれる：
- L: 位相0
- R: 位相π/3（60度ズレ）

### 時間同期パン

1.37秒マスターループ内で「パン位置 ＝ 時間位置 × π」のように、
空間位置を時間軸に連動させるとパン自体がリズムになる。

## 音の粒レシピ（コード不要・設計思想として）

> 粒＝音ではなく1リズムイベント。「顕微鏡作業」がAlva Noto/Ikedaの真髄。

### 共通準備

- 波形は1ms単位のグリッドで扱う
- ゼロクロス点（波形が0を横切る瞬間）を精密に特定してカット
- ゼロクロス以外の点でカットすると「プチッ」というクリック雑音が出る
- **エンベロープは直線的（0msアタック・0msディケイ）** — 曲線は使わない
- 粒間には **20ms以上の無音ギャップ** を必ず入れる
- 粒の配置は1ms精度で行うが、完全なグリッドには乗せない

### Alva Noto 粒レシピ

**材料**: サイン波（400Hz中心、200〜800Hzの範囲で選ぶ）+ テストトーンまたは環境ノイズ

**手順**:

1. 約1msのサイン波を生成する
2. ゼロクロス直前の立ち上がり部分（0.1〜0.3ms）をカットして先頭を整形する
3. 末尾の微細な残響部分（0.2ms程度）を別途カット
4. 波形シェイプで非線形倍音を付与（Chebyshev変換またはソフトクリップ）
5. L/Rで位相を30°ずらして保存

**仕上がり仕様**:
- 粒の長さ: 0.2〜0.8ms
- 配置間隔: 47〜72ms（ループの中で繰り返す）

### Ryoji Ikeda 粒レシピ

**材料**: 超高域サイン波（12〜18kHz）+ 純白ノイズの瞬間的な点

**手順**:

1. 0.5ms以下の超短いサイン波を生成する
2. 立ち上がりの最初の0.05msだけを切り取る（「電子の針」イメージ）
3. HPF（18kHz）と LPF（20kHz）で帯域を極限まで絞り純度を高める
4. 出現タイミングは非周期的（20〜180msのランダム間隔）
5. 8Hz前後のLFOでパンを往復自動化する

**仕上がり仕様**:
- 粒の長さ: 0.05〜0.15ms（人間の知覚限界以下）

### 共通仕上げ技法（両レシピ共通）

| 要素 | 内容 |
|------|------|
| エンベロープ | 直線のみ（アタック0ms・ディケイ0ms） |
| 無音ギャップ | 粒間20ms以上 |
| 配置精度 | 1ms精度で置くが完全グリッドは避ける |

### 設計思想の核心

「音」ではなく「粒」を作る。粒1つ＝リズムイベント1つ。
波形の「顕微鏡作業」（ゼロクロス・0.1ms精度の整形）がAlva Noto美学の核。
Ikedaは人間の知覚限界以下の短さを追求し、音というより「存在の痕跡」を置く。
