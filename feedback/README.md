# フィードバックシステム

## Claudeへの指示（セッション開始時に必ず読む）

1. `feedback/knowledge.md` を読み、過去の試行から得られた法則を把握する
2. 直近3セッション分のログ（`feedback/logs/` 内の新しいもの）を確認する
3. 今回の作業モジュールに関連する知識を優先的に参照する

---

## フィードバックセッションの進め方

### ループの1サイクル

```
① ユーザー：「〇〇な音にしたい」「ここが違う」と指示する
    ↓
② Claude：変更仮説を1行で述べてからコードを修正する
    ↓
③ ユーザー：音を聴く
    ↓
④ ユーザー：「良くなった」「変わってない」「悪くなった」＋一言コメント
    ↓
⑤ Claude：feedback/logs/YYYY-MM-DD.json に1エントリ追記する
    ↓（3回以上の試行のあと）
⑥ Claude：knowledge.md の関連セクションを更新する
```

### ログエントリの記録ルール

- ファイル名: `feedback/logs/YYYY-MM-DD.json`（1日1ファイル、配列として追記）
- 同日2回目以降は既存ファイルに追記（上書きしない）
- セッションIDは `YYYY-MM-DD-NNN`（NNNはその日の通し番号）

### knowledge.md の更新ルール

- 3回以上「良くなった」で確認されたパターンのみ「効いた操作」に昇格する
- 1回でも「悪くなった」は即記録する（警告として重要）
- 「変わってない」が2回続いたら、アプローチの見直しを提案する

---

## ファイル構成

```
feedback/
├── README.md          # このファイル（Claudeへの指示）
├── knowledge.md       # 蒸留された法則（Claudeが更新）
└── logs/
    ├── 2026-03-28.json
    ├── 2026-03-29.json
    └── ...
```

## ログのJSONスキーマ

```json
[
  {
    "session_id": "2026-03-28-001",
    "module": "grain_synth",
    "iteration": 1,
    "intent": "ユーザーが言葉で表現した意図",
    "hypothesis": "Claudeが立てた変更の仮説（1行）",
    "changes": [
      { "target": "ファイル名またはパラメーター名", "before": "変更前", "after": "変更後" }
    ],
    "result": "better | no_change | worse",
    "comment": "ユーザーの一言コメント（任意）",
    "timestamp": "ISO 8601"
  }
]
```
