## Max/MSPシステムの論理構造

### 非タイムライン型シーケンス
- DAWのような視覚的タイムラインを**徹底的に否定**
- ショーン・ブース「画面上にこれから何が来るか見えることは、聴覚的な判断を損なう」
- `phasor~`（オーディオレートのランプ信号）を基準時計として使用
- 論理演算・条件分岐でイベントを発生させる

### イベント駆動型ロジック
- リズムは「あるイベントが別のイベントを引き起こす」物理的な連鎖
- ボールが跳ね返る運動モデルに近い設計
- `coll`オブジェクトに格納されたデータやリアルタイム関数計算で次の音を決定

### スタイルの圧縮（Style Compression）
- アルゴリズムとは「自らの即興的ジェスチャーや音楽的好みを圧縮して再現可能にしたもの」
- 手動で行っていたマイクロ・エディットをスクリプト化
- わずかなパラメータ調整だけで無限のバリエーションを生成

### パラメータの比率ベース設計
- 定数をそのまま使わず、**すべてのパラメータが何らかの比率（Ratio）に基づいて動的に計算される**
- システム全体が相互依存した無数のモジュールの集合体

## 音響合成の高度なテクニック

### 1. 物理モデリング（Physical Modeling）

**デジタル導波管合成（Digital Waveguide Synthesis）：**
- 遅延線とフィルタを組み合わせ、管楽器・弦楽器の共鳴をモデル化

**質量-バネモデル（Mass-Spring Models）：**
- 物体の振動を「質量」と「バネ」として捉え、剛性・減衰をパラメータ化

```supercollider
// 物理モデル的なアプローチ例（Modalysイメージ）
SynthDef(\ae_modal, {
    var exc = Impulse.ar(1) * LFNoise0.kr(4).range(0.3, 1.0);
    var resonators = Mix.fill(8, {|i|
        var freq = (220 * (i + 1)) * LFNoise1.kr(0.1).range(0.98, 1.02);
        var decay = LFNoise0.kr(0.2).range(0.5, 4.0);
        Ringz.ar(exc, freq, decay)
    });
    Out.ar(0, resonators * 0.1);
}).add;
```

### 2. FMおよびフェーズ・モジュレーション

- `simpleFM~`のような基本モジュールを**数十〜数百重ね合わせ、相互変調**
- スタティックでありながら内部で絶えず変化する複雑な倍音構造
- Yamaha FS1Rのフォルマント合成的なFM手法 → 「声」のようなテクスチャの源泉

```supercollider
// FM相互変調の例
SynthDef(\ae_fm_complex, {
    var ops = Array.fill(6, {|i|
        var ratio = [1, 1.41, 2, 3.17, 4, 7][i];
        var index = LFNoise1.kr(0.05 * (i+1)).range(0.5, 8.0);
        SinOsc.ar(220 * ratio)
    });
    var mod = ops[0] * ops[1] * ops[2];
    var car = SinOsc.ar(220 + (mod * 440));
    Out.ar(0, car * 0.2);
}).add;
```

### 3. フィードバック・ディレイ・ネットワーク（FDN）

単純なリバーブではなく**FDN（複数の遅延線をマトリックスで接続し相互フィードバック）**を多用。

| FDNの構成要素 | 技術的役割 | Autechre的応用 |
|--------------|-----------|----------------|
| 遅延線 | 反射の時間間隔を決定 | **素数や無理数に基づいた遅延長**を設定し定在波を回避 |
| フィードバック行列 | 信号の混合と減衰の制御 | ユニタリ行列でエネルギーを保存しつつ複雑な共鳴を得る |
| 吸収フィルタ | 周波数ごとの減衰率を調整 | 物理的な部屋の素材（木・金属・布）をシミュレート |

## リズムの数学的設計

### スウィングのランダム化
- スウィング量が一定ではなく、**音ごとに異なる量でランダムに適用**
- これがAutechre特有の緊張感の源

**再現ロジック：**

1. **多層的量子化**：8分・16分・非グリッドなど、異なる量子化レイヤーを同時に走らせランダムにスイッチ
2. **確率的ゲート**：マルコフ連鎖やユークリッド・アルゴリズムに基づいてビート発生を確率的に決定
3. **アンプエンベロープの動的制御**：リリースを極端に長く設定し、音の重なりでリズムをテクスチャ化

### ユークリッド・リズム（E(k, n)）

$$E(k, n)$$

- $k$：音の数、$n$：全ステップ数
- 一定のステップ中に音を可能な限り均等に配置する数学的手法
- 複数のユークリッドリズムを重ね、各速度・位相を動的に変化させる

## ミキシングと後処理の哲学

### サイドチェーンEQトリック（ショーン・ブース公開）
- 通常のマルチバンド・コンプレッサーは**位相崩れを嫌うため使わない**
- コンプのサイドチェーン・フィルタで **3kHz・6kHz付近に強いピーク** を設定
- 人間の耳が最も敏感な帯域だけを的確に抑制
- → 「耳に優しいが密度の高い」サウンド

### ビットクラッシングとADATエミュレーション
- Juno-106などのアナログシンセを**1990年代の44.1kHzビットクラッシャーに通す**
- 当時のAD/DAコンバーター特有のキャラクターを付加
- ビットクラッシャーをメイン出力に適用してDATテープへの録音をシミュレート
- → 全体の音響的まとまりを得る

### オシロスコープによる視覚的監視
- Philips製オシロスコープを常にスタジオに設置
- 波形の形そのものが美しく見えるようなトラックを作る
- 音と視覚の融合的アプローチ

## 現代のプラグイン環境での再現

### Fors（Max for Live）
- Autechreの制作手法を最も深く理解しているデベロッパー

| デバイス名 | 特徴 |
|-----------|------|
| **Opal** | リズムマシン・ドラムシンセ。確率論的トリガー・ラチェット・パラメータロック |
| **Chiral** | フェーズシェイピング合成。サイン波を極限まで歪め、透明なデジタルトーン〜グリッチまで |
| **Mass** | モーダル合成シンセ。ベル・撥弦・打楽器的な物理モデリングサウンド |

### Ableton Live 12でのワークフロー

**SQシーケンサーの多用：**
- スケールロックされたピッチ・ランダムなオクターブシフト・ベロシティマッピング・ラチェット
- 異なるトラック長（例：7ステップ × 5ステップ）で走らせ → 永遠に変化し続けるシーケンス

**ベロシティ・モジュレーションの極大化：**
- Operator（FMシンセ）のフィルタ・カットオフ・デチューン・エンベロープタイムをベロシティにマッピング
- 一音ごとに音色が激しく変化する「音の粒子」を生成

**LFO 3.1による確率制御：**
- バイナリ波形のLFOでエフェクトのON/OFFを確率的にスイッチ
- Beat RepeatやEchoをこの方法でトリガー → 制御されたカオス

## Autechreの核心思想（実装への示唆）

1. **機材への依存を捨て、システムを構築する**
   - 単一のプリセットを使わず、Max/SC内で「制作ルール（アルゴリズム）」を策定する

2. **非線形な時間を導入する**
   - 4/4拍子の固定観念を捨て、確率論的シーケンス・ポリリズム・スウィングのランダム化

3. **音響合成の深層へ向かう**
   - FM合成と物理モデリングを主軸に、音の一粒一粒にマイクロ・モジュレーションを施す

4. **物理的な感性を忘れない**
   - テープ編集・ミキサーのEQ操作のような「音を切り刻み、極端に加工する」荒々しい姿勢を維持

「再現の真髄は完成された音を真似ることではなく、自分自身をも驚かせるような予測不能なフィードバック・ループの中に身を置くことにある」
